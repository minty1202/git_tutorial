$ git init をすると .gitディレクトリが作成される ローカルリポジトリのこと
gitで必要なほとんどのファイルが作られる

$ git clone <リポジトリ名> gitリポジトリのコピーを作成する 他の人がすでに作成しているプロジェクトを始める ファイルがワークツリー .gitディレクトリにコピーされる

$ git add <ファイル名>
$ git add <ディレクトリ名>
$ git add .
変更をGitのステージに追加する ステージはコミットをする準備をする

$ git commit
$ git commit -m "<メッセージ>" # Gitのエディターが立ち上がらずcommitできる
$ git commit -v # ファイルのどんな変更をしたのか変更内容を確認してcommitできる
メッセージ付きで記録する なんのために記録するのかをわかるようにするため
commitメッセージはわかりやすい言葉にする
一行目 変更内容の要約
二行目 空白
三行目 変更した理由
